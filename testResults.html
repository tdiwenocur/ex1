Test Results

Wenocur, Tamar Des 157

The audience testing was a real eye-opener on all the things that were working (and could use improvement) in my interface. I was using my interface in the ways that I knew would work, but the audience-testing literally tested every corner and bug in my interface and brought the holes into the limelight.

Luckily, the things that I thought worked did work and were pretty easy to find. Both testers were able to work the zoom function on the map, work the search function, use the legend, and go to the list of buildings index page. My users did not struggle to figure out what the function of my interface was. They both said it was pretty clear from the get-go.

The issues they had were also pretty abundant. I didn't flesh out the whole map legend, so they expected all the features to be visible. There were also some technical issues when they tried to zoom at the same time as the legend was triggered.

I think my design was successful in making people think that it was a real web-app. Making the prototype not completely functional is a downside as well as an upside. I think a pitfall of making a realistic-looking interface is that users expect it being fully functional.
